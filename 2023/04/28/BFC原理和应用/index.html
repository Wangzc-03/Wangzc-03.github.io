BFC原理和应用

1、BFC是什么？

BFC（block formatting context）直译为“块级格式化上下文”

MDN给出的解释是：BFC是Web页面的可视CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。

通俗来讲：BFC是一块独立的布局区域，BFC区域的元素有自己的布局规则，不受外部的影响。

2、BFC区域规则

  1. BFC区域不会和浮动区域重叠

  2. BFC内部的盒子会在垂直方向上依次排列

  3. 计算BFC区域高度时，浮动的子元素也参与计算

  4. 每个盒子的左外边框紧挨着包含块的左边框，即使浮动元素也是如此。

  5. BFC内部盒子垂直方向上的距离由margin值决定，同一个BFC中的相邻两个盒子之间的margin值会重叠

3、如何触发BFC

  1. overflow: auto/ hidden;

  2. position: absolute/ fixed;

  3. float: left/ right;

  4. display: inline-block/ table-cell/ table-caption/ flex/ inline-flex

    还有许多方法，这里不过多介绍

4、BFC的一些应用场景

  1. 清除浮动

    当子元素浮动 父元素会失去高度，给父元素添加overflow：hidden属性

    给父盒子添加overflow：hidden属性后

  2. 解决边距重叠

    同级盒子：在标准流文档中 ，两个盒子左右排列时边距不会重叠，上下排列时会发生边距重叠。

    本来两者之间应该有30px的距离，但检查时会发现只有二十，上边的10px和下边的20px发生重叠

    给下边的盒子套上一个父盒子，并触发该父盒子的BFC，使得里面的布局独立出来就可以解决

    此时两盒子之间的距离才是30px

  3. 两栏的自适应布局

    给固定盒子设置宽度并浮动，给非浮动盒子开启BFC

    此时非浮动盒子不会被压住而是贴着固定盒子并且自适应宽度显示

  4. 解决嵌套块元素塌陷问题

    在标准流文档中，想要让子盒子距离父盒子一段距离时，直接给子盒子添加margin会出现下面的问题

    本应该出现在子盒子和父盒子之间的边距消失了，反而父元素的边距被改变了，此时父子盒子会共享最大的那个边距。

    给父盒子添加overflow：hidden属性，触发BFC使得盒子的布局独立出来。

    该问题也可以通过给父盒子border或者padding-top值来解决（但是这两种都会影响到布局，不推荐）
